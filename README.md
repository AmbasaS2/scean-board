# 🧭 Scene Board

**Scene Board는 캐릭터 응답 하단에 출력된 상태창을 본문에서 분리해 메시지 아래의 별도 카드로 만들어 주는 SillyTavern 확장입니다.**

---

## ⚙️ 확장 설정

**Extensions → Scene Board**에서 기본 설정을 바꿀 수 있습니다.

| 설정 | 설명 |
|---|---|
| **Scene Board 활성화** | 새 캐릭터 응답 하단의 상태창을 감지해 카드로 분리합니다. |
| **자동 줄바꿈** | 한 줄에 붙어 출력된 여러 항목을 줄 단위로 정리합니다. |
| **채팅 번호 숨기기** | 카드에 표시되는 `#번호`를 숨깁니다. |
| **씬보드 글씨 크기** | 분리된 카드와 보관함의 글씨 크기를 조절합니다. |

---

## ✂️ 상태창 분리

Scene Board는 **최신 캐릭터 응답의 마지막 부분**에서 시간이나 장소를 중심으로 상태창 형식을 찾습니다.

### 캐릭터가 출력한 응답

````text
그는 창밖을 확인한 뒤 조용히 문을 닫았다.

```info board
🕰️ 늦은 저녁
📍 기숙사 복도
등장인물: A, B
👗 복장: 교복과 검은 망토
Mood: tense but quiet
소지품: 지팡이, 편지
Soundtrack: 창문을 두드리는 빗소리
```
````

### 카드로 분리되는 영역

```text
🕰️ 늦은 저녁
📍 기숙사 복도
등장인물: A, B
👗 복장: 교복과 검은 망토
Mood: tense but quiet
소지품: 지팡이, 편지
Soundtrack: 창문을 두드리는 빗소리
```

상태창으로 감지된 영역만 본문에서 빠지고, RP 문장 아래에 별도의 Scene Board 카드로 표시됩니다.

### 감지하는 형식

- 시간은 `Time`, `시간`, `날짜`, `🕰️`, `⏰`처럼 표시할 수 있습니다.
- 장소는 `Location`, `장소`, `위치`, `📍`, `🧭`처럼 표시할 수 있습니다.
- 시간과 장소를 함께 넣으면 가장 안정적으로 감지합니다. 둘 중 하나만 사용하는 경우에는 등장인물, 복장, 소지품, 분위기처럼 다른 항목을 함께 적어주세요.
- `Soundtrack`, `Affection`, `Secret`처럼 사용자가 직접 만든 항목도 `항목명: 내용` 형식이면 함께 표시됩니다.

일반 텍스트와 코드 블록을 지원하며, `scene board`, `info board`, `info panel`, `status panel`, `HUD` 등의 코드 블록 이름과 `<Info_panel>` 형식을 인식합니다.

---

## 💬 분리된 Scene Board 카드

분리된 카드는 최신 캐릭터 메시지 아래에 표시됩니다.

| 버튼 | 기능 |
|---|---|
| **저장** | 현재 카드를 보관함에 저장합니다. 같은 카드는 중복 저장하지 않습니다. |
| **저장됨** | 이미 보관함에 저장된 카드라는 표시입니다. |
| **삭제** | 현재 카드를 최근 목록에서 제거합니다. |
| **‹ / ›** | 같은 캐릭터의 최근 카드 사이를 이동합니다. |
| **#번호** | 카드가 분리된 채팅 메시지 번호를 표시합니다. 설정에서 숨길 수 있습니다. |

최근 카드는 캐릭터별로 **최대 3개**까지 유지됩니다. 오래 보관할 카드는 **저장**을 눌러 보관함에 남길 수 있습니다.

---

## 🗂️ 보관함

확장 메뉴에서 **Scene Board**를 누르면 저장한 카드를 모아볼 수 있습니다.

| 위치 | 기능 |
|---|---|
| **전체** | 모든 캐릭터의 저장한 카드를 함께 봅니다. |
| **캐릭터 이름** | 선택한 캐릭터의 저장한 카드만 봅니다. |
| `📋` | 카드 내용을 복사합니다. |
| `🗑️` | 저장한 카드 하나를 삭제합니다. |
| `⚙` | 글씨 크기와 초기화 메뉴를 엽니다. |
| `×` | 보관함을 닫습니다. |

---

## 🧠 RP 반영

가장 최근에 분리된 카드는 현재 장면의 참고 정보로 메인 RP에 자동 반영됩니다.

- 시간과 날짜
- 장소와 배경
- 등장인물
- 소지품
- 날씨와 분위기
- 관계, 목표, 기타 장면 정보

**Scene Board 활성화**를 끄면 이미 카드로 분리된 상태창은 RP에 반영되지 않습니다. 다시 활성화하면 다음 새 캐릭터 응답부터 상태창을 감지합니다.

---

## 💾 저장과 초기화

| 구분 | 저장 방식 |
|---|---|
| **최근 카드** | 캐릭터별 최대 3개를 유지하며, 메시지 아래의 `‹ / ›`로 확인합니다. |
| **저장한 카드** | 직접 삭제하거나 초기화할 때까지 보관함에 남습니다. |

보관함의 `⚙`에서 아래 항목을 사용할 수 있습니다.

| 기능 | 설명 |
|---|---|
| **현재 캐릭터 씬보드 초기화** | 현재 캐릭터의 최근 카드와 저장한 카드를 모두 삭제합니다. |
| **씬보드 내역 전체 초기화** | 모든 캐릭터의 최근 카드와 저장한 카드를 모두 삭제합니다. |

초기화한 카드는 되돌릴 수 없습니다.

---

## ⚠️ 참고사항

- Scene Board는 상태창을 출력하는 확장이 아닙니다. 캐릭터나 프리셋이 응답 하단에 상태창을 출력해야 작동합니다.
- Scene Board를 설치하고 활성화한 뒤, 프리셋의 양식에 따라 출력된 **새 캐릭터 응답 하단의 상태창**을 파싱합니다. 이전 채팅의 상태창은 카드로 분리되지 않습니다.
- 상태창이 응답의 상단이나 중간에 있거나, 정규식으로 별도 꾸며져 있으면 카드로 파싱되지 않을 수 있습니다.
- 본문과 상태창의 경계가 불분명하거나 항목 형식이 너무 자유로우면 원문을 그대로 유지합니다.
- 번역기나 다른 확장에서 Scene Board 내용까지 읽으려면 메시지 객체의 `msg.extra.sceneBoard.text`를 함께 처리해야 합니다. 화면에 표시된 카드는 `.sb-inline-panel`에서 찾을 수 있습니다.

```js
const sceneBoardText = msg?.extra?.sceneBoard?.text || '';
const sceneBoardCards = document.querySelectorAll('.sb-inline-panel');
```
